## Введение

ElemWowMap — это интерактивный виджет карты для платформы Insight, поддерживающий несколько режимов визуализации, включая теплокарты, маркеры точек, хороплеты и анимации на основе времени. Это руководство поможет вам настроить и сконфигурировать виджет ElemWowMap в вашей среде Insight.

## Предварительные требования

Перед настройкой ElemWowMap убедитесь, что у вас есть:

- **Node.js** версии 18 или выше
- **npm** версии 9 или выше
- Доступ к платформе Insight
- Базовое понимание файлов конфигурации JSON
- Набор данных с географическими данными (широта/долгота или коды регионов)

## Регистрация виджета

Чтобы сделать ElemWowMap доступным в редакторе Insight, необходимо зарегистрировать его в файлах конфигурации.

### Шаг 1: Конфигурация платформы

Сначала настройте платформу для распознавания вашего пакета виджетов. Отредактируйте `public/config/platform/widgets.local.json`:

```json
{
  "widgets": {
    "packages": {
      "vendor": {
        "url": "./widgets.local/vendor"
      }
    }
  }
}
```

### Шаг 2: Конфигурация пакета

Зарегистрируйте ElemWowMap в `public/config/package/widgets.local.json`:

```json
{
  "elements": [
    {
      "type": "vendor::ElemWowMap",
      "name": "Карта",
      "icon": "mdi-map"
    }
  ],
  "tools": {
    "page": [
      {
        "name": "Визуализации",
        "elements": [
          "vendor::ElemWowMap"
        ]
      }
    ],
    "template": [
      {
        "name": "Визуализации",
        "elements": [
          "vendor::ElemWowMap"
        ]
      }
    ]
  }
}
```

### Объяснение полей конфигурации

- **`type`**: Идентификатор виджета `vendor::ElemWowMap`
- **`name`**: Отображаемое имя в палитре виджетов
- **`icon`**: Имя иконки Material Design Icons (например, `mdi-map`)
- **`tools.page`**: Виджеты, доступные при редактировании страниц
- **`tools.template`**: Виджеты, доступные при редактировании шаблонов

## Файлы конфигурации

### Дескриптор виджета

Виджет ElemWowMap имеет файл `descriptor.js`, который определяет:

- **Props**: Настраиваемые свойства (привязка полей, цвета, настройки)
- **Dataset Contract**: Требуемые поля набора данных и операции
- **Panels**: Панели конфигурации, доступные в редакторе
- **CSS Variables**: Опции настройки темы

Ключевые свойства дескриптора включают:

```javascript
export const descriptor = () => ({
    props: {
        // Привязка полей
        latitudeField: {
            type: String,
            default: 'latitude',
            label: 'Поле широты'
        },
        longitudeField: {
            type: String,
            default: 'longitude',
            label: 'Поле долготы'
        },
        valueField: {
            type: String,
            default: 'value',
            label: 'Поле значения'
        },
        // ... другие свойства
    },
    dataset: {
        vars: {
            dimension: { operation: StoreOperation.ALL }
        }
    }
});
```

### Конфигурация панелей

ElemWowMap предоставляет несколько панелей конфигурации:

- **Панель набора данных**: Выбор и настройка источников данных, полей карты
- **Панель визуализации**: Выбор режима визуализации (теплокарта, точки, хороплет)
- **Панель теплокарты**: Настройка радиуса, размытия, интенсивности
- **Панель хороплета**: Настройка цветовых схем, методов агрегации
- **Панель анимации**: Настройка анимаций на основе времени
- **Панель фильтров**: Настройка фильтров по годам и категориям
- **Панель цветов**: Настройка цветов категорий
- **Панель пользовательских стилей**: Расширенная настройка CSS

## Режимы визуализации

### Режим теплокарты

Визуализирует плотность данных с помощью интенсивности цвета:

**Настройка:**
- **Радиус**: Размер точек тепла (по умолчанию: 25)
- **Размытие**: Количество размытия (по умолчанию: 15)
- **Минимальная прозрачность**: Минимальная прозрачность (0-1)
- **Максимальная интенсивность**: Максимальная интенсивность цвета
- **Минимальная интенсивность**: Минимальная интенсивность цвета

**Применение:**
- Плотность населения
- Частота событий
- Горячие точки активности

### Режим точек

Отображает отдельные точки данных как маркеры:

**Настройка:**
- **Радиус точки**: Размер точек (по умолчанию: 5)
- **Прозрачность точки**: Прозрачность (0-1)
- **Размер точки**: Масштабный коэффициент
- **Цветовая схема**: Цветовое отображение для категорий
- **Кластеризация**: Включить/отключить кластеризацию маркеров

**Применение:**
- Отдельные местоположения
- Маркеры точек интереса
- Местоположения событий

### Режим хороплета

Раскрашивает регионы на основе агрегированных данных:

**Настройка:**
- **SVG файл карты**: Файл карты для использования (например, `ru.svg`)
- **Поле кода региона**: Поле, содержащее идентификаторы регионов (формат RU-XXX)
- **Цветовая схема**: Цветовая палитра (например, YlOrRd, Blues, Greens)
- **Цветовые шаги**: Количество цветовых градаций
- **Метод агрегации**: Как агрегировать значения (сумма, среднее, максимум, минимум)
- **Цвет отсутствующих данных**: Цвет для регионов без данных

**Применение:**
- Региональная статистика
- Административные данные
- Географические сравнения

### Режим анимации

Визуализация на основе времени с эффектами частиц:

**Настройка:**
- **Поле даты**: Поле, содержащее даты/временные метки
- **Скорость анимации**: Скорость воспроизведения
- **Автовоспроизведение**: Автоматически запускать анимацию
- **Время затухания**: Как долго частицы остаются видимыми (в днях)
- **Длительность свечения частиц**: Длительность эффекта свечения (мс)

**Применение:**
- Анализ временных данных
- Прогрессия событий
- Визуализация временных рядов

## Типичные сценарии настройки

### Сценарий 1: Базовая настройка теплокарты

1. **Зарегистрируйте виджет** в `widgets.local.json`:
```json
{
  "type": "vendor::ElemWowMap",
  "name": "Карта",
  "icon": "mdi-map"
}
```

2. **Настройте поля набора данных** в панели набора данных:
   - Выберите поле широты (например, `lat`)
   - Выберите поле долготы (например, `lng`)
   - Выберите поле значения для агрегации (например, `count`)

3. **Выберите режим теплокарты** в панели визуализации

4. **Настройте параметры теплокарты**:
   - Установите радиус: 25-50
   - Установите размытие: 15-25
   - Отрегулируйте диапазон интенсивности

5. **Настройте внешний вид**:
   - Установите провайдера карт (OpenStreetMap по умолчанию)
   - Настройте начальный уровень масштабирования
   - Включите/отключите элементы управления

### Сценарий 2: Хороплет с регионами

1. **Подготовьте ваш набор данных**:
   - Включите поле кода региона (формат: `RU-XXX`)
   - Включите поле значения для агрегации
   - Опционально включите поле названия региона

2. **Настройте поля**:
   - Установите `regionCodeField` на вашу колонку кода региона
   - Установите `valueField` на вашу колонку метрики
   - Установите `svgMapFile` на `ru.svg` (или вашу пользовательскую карту)

3. **Выберите режим хороплета**

4. **Настройте хороплет**:
   - Выберите цветовую схему (например, YlOrRd)
   - Установите цветовые шаги (рекомендуется 5-7)
   - Выберите метод агрегации (сумма, среднее и т.д.)
   - Установите цвет отсутствующих данных

5. **Включите детализацию** (опционально):
   - Установите `enableDrillDown` в true
   - Настройте начальный уровень детализации

### Сценарий 3: Анимированный временной ряд

1. **Подготовьте набор данных** с полем даты

2. **Настройте поля**:
   - Установите поля широты и долготы
   - Установите поле даты для анимации
   - Установите поле значения

3. **Выберите режим точек или теплокарты**

4. **Настройте анимацию**:
   - Включите `autoPlay`
   - Установите `animationSpeed` (1-10)
   - Настройте `fadeTime` (дни, в течение которых частицы остаются видимыми)
   - Установите `particleGlowDuration` (мс)

5. **Добавьте фильтры** (опционально):
   - Включите фильтр по годам
   - Включите фильтр по категориям
   - Настройте параметры фильтров

### Сценарий 4: Пользовательская SVG карта

1. **Подготовьте файл SVG карты**:
   - Убедитесь, что элементы SVG имеют атрибуты `id`, соответствующие кодам регионов
   - Поместите файл SVG в `src/ElemWowMap/assets/` или укажите URL

2. **Настройте пользовательскую карту**:
   - Установите `customSvgUrl` на путь/URL вашего файла SVG
   - Установите `svgIdField` для соответствия полю идентификатора вашего набора данных
   - Настройте `useMapNameForHover`, если используете названия элементов карты

3. **Настройте хороплет**:
   - Выберите режим хороплета
   - Сопоставьте коды регионов с ID элементов SVG
   - Настройте цветовую схему

## Решение проблем

### Виджет не отображается в палитре

**Проблема**: ElemWowMap не появляется в палитре редактора.

**Решения**:
1. Проверьте, что синтаксис `widgets.local.json` является валидным JSON
2. Убедитесь, что тип виджета точно `vendor::ElemWowMap`
3. Проверьте, что URL пакета правильный в конфигурации платформы
4. Перезапустите сервер разработки (`npm start`)
5. Очистите кэш браузера

### Карта не отображается

**Проблема**: Карта показывает пустое место или ошибку.

**Решения**:
1. Проверьте консоль браузера на наличие ошибок JavaScript
2. Убедитесь, что URL провайдера карт доступен
3. Проверьте настройки CORS, если используете пользовательский провайдер тайлов
4. Убедитесь, что библиотека Leaflet загружена
5. Проверьте, что контейнер карты имеет размеры

### Набор данных не загружается

**Проблема**: Виджет показывает "Нет данных" или ошибку загрузки.

**Решения**:
1. Проверьте, что поля набора данных соответствуют требованиям виджета:
   - Поле широты существует и содержит валидные координаты
   - Поле долготы существует и содержит валидные координаты
   - Поле значения существует (для агрегации)
2. Убедитесь, что имена полей правильные (с учетом регистра)
3. Проверьте, что набор данных содержит данные для выбранного диапазона дат
4. Проверьте консоль браузера на наличие ошибок API
5. Убедитесь в правах доступа к набору данных

### Теплокарта не отображается

**Проблема**: Слой теплокарты не появляется.

**Решения**:
1. Проверьте, что поля широты/долготы содержат валидные числовые значения
2. Убедитесь, что поле значения имеет числовые данные
3. Отрегулируйте настройки радиуса и размытия теплокарты
4. Увеличьте максимальную интенсивность, если теплокарта слишком бледная
5. Проверьте консоль браузера на наличие ошибок Leaflet.heat

### Хороплет не раскрашивает регионы

**Проблема**: Регионы появляются, но не раскрашены.

**Решения**:
1. Проверьте, что поле кода региона соответствует ID элементов SVG
2. Убедитесь, что формат кода региона правильный (должен соответствовать шаблону `RU-XXX`)
3. Проверьте, что поле значения имеет данные для регионов
4. Убедитесь, что метод агрегации подходящий
5. Проверьте, что файл SVG карты загружен правильно
6. Убедитесь, что конфигурация `svgIdField` соответствует вашим данным

### Анимация не работает

**Проблема**: Анимация не воспроизводится или частицы не появляются.

**Решения**:
1. Проверьте, что поле даты выбрано и содержит валидные даты
2. Убедитесь, что формат даты соответствует ожидаемому формату
3. Включите `autoPlay` или используйте кнопку воспроизведения
4. Отрегулируйте настройки скорости анимации
5. Проверьте, что `fadeTime` не слишком короткий
6. Убедитесь, что набор данных имеет несколько временных точек

### Проблемы с производительностью

**Проблема**: Карта работает медленно или не отвечает.

**Решения**:
1. Уменьшите размер набора данных (добавьте фильтры)
2. Включите кластеризацию точек для больших наборов данных точек
3. Увеличьте размытие теплокарты для снижения нагрузки на рендеринг
4. Отключите ненужные анимации
5. Уменьшите количество видимых частиц
6. Проверьте консоль браузера на наличие утечек памяти
7. Используйте серверную фильтрацию, когда возможно

### Стили не применяются

**Проблема**: Пользовательские CSS стили не отображаются.

**Решения**:
1. Проверьте специфичность селекторов CSS
2. Убедитесь в изоляции пространства имен (`.elemwowmap-widget-root`)
3. Проверьте, что стили ограничены контейнером виджета
4. Проверьте наличие синтаксических ошибок CSS
5. Убедитесь в правильности конфигурации панели пользовательских стилей
6. Проверьте, что стили не переопределены темой
